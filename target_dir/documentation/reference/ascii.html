控制字符的常见用途扩展（ASCII 码 0到31）

ASCII 概要
0空字符，通常 （特别是在 PureBasic） 用于指示字符串的结尾。
1标题开始。表示数据块或数据块的标头部分的开始。
2正文开始。通常情况下在通信过程中发送的文本块的第一个字符。
3正文末尾通常情况下在通信过程中发送的文本块的最后一个字符。
4标题结尾。用于表示传输结束。
5查询。为了得到一个响应发送一请求查询。
6确认。由一个接收器发送，表示它已经收到或理解查询请求。
7报警. 发出噪音（通常是PC扬声器“滴”声）。
8退格。将光标向左移动一个位置（也可能删除左侧的一个字符）。
9水平制表符. 水平制表符，通常用于设置文本格式。
10换行符。用来表示在打字机或文本模式打印或显示跳转到新的一行。通常用作文本行的结尾部分.
11垂直制表符。与（水平）标签类似，但它是向下移动一行，而不是水平移动一列。
12走纸控制。通常用此字符告诉 打印机(文本模式) 向下移动一个页面。
13回车符。它表示的打字机或打印机头搬回行的起始处的字符。通常用作行尾的字符。 通常用作文本行的结尾部分.
14移位输出。其含义是字符块的开始。
15移位输入。关闭上述类型的块的传输。
16数据链路转义。用于指示以下控制字符应被解释为数据，而不是控制字符。
17设备控制 1. 通常用来打开一个设备。最常见的用途在软件流控制串行通信中表示 XON 字符。
18设备控制 2. 另一种情况依赖设备控制字符。
19设备控制 3. 通常用于关闭一台设备。最常见的用途是作为软件流控制的串行通信中XOFF字符。
20设备控制 4. 另一种情况依赖设备控制字符。
21否定。通常用来表示未接收到的或不确定（错误的）。
22闲置同步。顾名思义，信号发送了同步(在定期时间) 以指示通信信道是空闲的但是仍然活跃。
23传输块结束。用来控制数据的传输通过指示块的结尾。这不与 EOT 相同。
24取消。通常是指以前发送的数据应该忽略，虽然具体的应用取决于应用程序。
25纸尽、媒体结束。用于表示一些媒体，例如磁带驱动器播放结束。
26换置、替补。用来表示一个字符已被取代的占位符字符。
27转义字符、换码、退出。字符键盘上用"转义序列"中按 Esc 键的制作通常提供的基于文本的格式设置信息显示（控制台、 打印机等）。
28文件分隔符
29组分隔符
20请求发送、记录分隔符
