UserGuide - Input & Output

每个 PureBasic 应用程序都可以用不同的方式与用户进行交互。

我们对不同方式之间进行区分
</br2>a) 只是输出信息
b)PureBasic应用与用户的交互，当用户输入后将结果再次输出。

很多年前，在没有图型用户接口(GUI)的DOS操作系统中，可以使用一个简单的“PRINT”命令向屏幕输出一些信息. 但是，今天当你使用像 Windows、 Mac OSX 或 Linux 操作系统时，这种 GUI 是总是存在，此时再使用简单的“PRINT”输出是不行的。

不同的输出情形我们有不同的输出方法:
- Debug 调试窗口 （只在 PureBasic 编程时使用）
- MessageRequester() (输出简短的文本信息到请求窗口)
- Files (用于将结果保存在一个文本文件中，等等.)
- Console控制台 (用于简单和几乎无图形的文本输出，非常类似于早些时候的DOS 时代.)
- Windows and gadgets (标准的 windows 桌面操作系统，例如应用程序的 GUI 元素)
- Screen (文本和图形直接输出在屏幕上，例如游戏)

PureBasic应用程序能够记录和处理用户的输入 , 这三个最后提到的输出选项也有可能获得用户输入:
- 在控制台 console 使用 Input()
- 在 window 窗口使用 WaitWindowEvent() / WindowEvent(), 可以获取window窗口中发生的事件 , 例如点击一个 button按钮 或者 在 StringGadget()字符串组件中输入文本.
- 在 graphics screen 图形屏幕中使用 keyboard
-以及还有可能获取用户输入的InputRequester()
